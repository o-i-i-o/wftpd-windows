# 测试脚本修复说明

## 2026-03-28 修复内容

### 问题描述

在 SFTP 测试过程中发现逻辑 BUG：**文件被提前删除，导致后续用例失败**

### 根本原因

1. **`test_rename()` 测试可能失败**
   - 如果重命名失败，文件名可能是 `test_renamed.txt` 而不是 `test_upload.txt`
   - 导致后续的 `test_symlink()` 和 `test_readlink()` 因为找不到目标文件而失败

2. **测试用例之间的依赖关系处理不当**
   - 多个测试用例依赖同一个文件 `test_upload.txt`
   - 前面的测试失败会影响后续所有依赖该文件的测试

3. **FTP 测试也存在类似问题**
   - `test_delete()` 总是尝试删除 `test_upload.txt`
   - 如果 `test_rename()` 成功执行，文件名已改为 `test_renamed.txt`
   - 导致 `test_delete()` 失败或无法正确清理

### 修复方案

#### 1. SFTP `test_rename()` 修复

```python
def test_rename(self):
    # ✓ 检查文件是否存在，不存在则重新上传
    try:
        self.sftp.stat(old_name)
    except FileNotFoundError:
        # 重新上传文件
        local_file = os.path.join(tempfile.gettempdir(), 'test_rename_upload.txt')
        with open(local_file, 'w', encoding='utf-8') as f:
            f.write('rename test content')
        self.temp_files.append(local_file)
        self.sftp.put(local_file, old_name)
    
    self.sftp.rename(old_name, new_name)
    
    # 验证新名称存在
    if exists:
        self.result.add_pass(test_name)
    else:
        self.result.add_fail(test_name, "重命名后文件不存在")
        return  # ✓ 文件不存在，跳过恢复原名，避免影响后续测试
        
    # 恢复原名
    try:
        self.sftp.rename(new_name, old_name)
    except:
        pass
```

**改进点**：
- ✓ 自动检查并重新创建所需文件
- ✓ 失败时跳过恢复原名，避免二次错误
- ✓ 使用独立的临时文件，不影响其他测试

#### 4. SFTP `test_remove()` 修复

def test_remove(self):

    possible_files = [
        self.test_dir + '/test_upload.txt',
        self.test_dir + '/test_renamed.txt'
    ]
    
    remote_file = None
    for file in possible_files:
        try:
            self.sftp.stat(file)
            remote_file = file

        except FileNotFoundError:
            continue

        remote_file = self.test_dir + '/test_delete.txt'

#### 5. FTP `test_delete()` 修复

```python
def test_delete(self):
    # ✓ 尝试多个可能的文件名
    possible_files = ['test_upload.txt', 'test_renamed.txt']
    remote_file = None
    
    files = self.ftp.nlst()
    for file in possible_files:
        if file in files:
            remote_file = file
            break
    
    if remote_file is None:
        # ✓ 如果没有找到文件，重新上传一个用于测试
        remote_file = 'test_delete.txt'
        test_content = f'Delete test content - {datetime.now().isoformat()}'
        local_file = os.path.join(tempfile.gettempdir(), 'test_delete_upload.txt')
        with open(local_file, 'w', encoding='utf-8') as f:
            f.write(test_content)
        self.temp_files.append(local_file)
        
        with open(local_file, 'rb') as f:
            self.ftp.storbinary(f'STOR {remote_file}', f)
    
    self.ftp.delete(remote_file)
    # 验证删除...
```

**改进点**：
- ✓ 智能查找可能存在的文件
- ✓ 自动创建测试文件
- ✓ 确保测试总能执行

### 修复效果

#### 修复前的问题流程
```
test_put_file() → 创建 test_upload.txt ✓
test_rename() → 重命名为 test_renamed.txt ✓
test_symlink() → 查找 test_upload.txt ✗ (文件不存在，失败)
test_readlink() → 查找 test_upload.txt ✗ (文件不存在，失败)
test_remove() → 查找 test_upload.txt ✗ (文件不存在，失败)
```

#### 修复后的流程
```
test_put_file() → 创建 test_upload.txt ✓
test_rename() → 重命名为 test_renamed.txt，然后恢复为 test_upload.txt ✓
test_symlink() → 检查文件存在 ✓ / 不存在则重新上传 ✓
test_readlink() → 检查文件存在 ✓ / 不存在则重新上传 ✓
test_remove() → 智能查找文件 (test_upload.txt 或 test_renamed.txt) ✓
```

### 关键改进

1. **自给自足的测试用例**
   - 每个测试都检查所需的前置条件
   - 如果前置条件不满足，自动创建
   - 不再依赖其他测试的执行结果

2. **容错性增强**
   - 即使前面的测试失败，后续测试也能正常执行
   - 测试结果更准确，不会因为依赖关系产生连锁失败

3. **资源管理优化**
   - 每个测试使用独立的临时文件
   - 所有临时文件都被跟踪并在 cleanup 中清理
   - 不会留下垃圾文件

4. **智能文件查找**
   - 考虑了多种可能的文件名
   - 自动适应前面测试的结果
   - 提高测试的鲁棒性
