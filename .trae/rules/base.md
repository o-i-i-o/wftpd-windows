- 这是一个针对win10/11的Rust项目，用于管理ftp/sftp服务器的配置和服务。
- rust egui框架负责视图层的实现，提供用户界面和交互功能管理配置和服务。后台程序负责业务逻辑的处理和数据存储。

# 项目规则
- 所有文件都必须使用UTF-8编码
- 能简化代码一定不要引入冗余代码
- 每次修改代码前，必须先阅读cargo.toml文件，学习组件版本文档和api使用方法
- 新增组件使用cargo add 组件名称 添加到cargo.toml文件中以避免引入错误版本的组件
- 每次修改代码后，必须执行cargo clippy检查并修复问题，
- **代码规范：** 
    - 不要试图用 #[allow(...)] 忽略错误！
    - 不要试图使用_ 来忽略未使用的变量或函数参数。要么真实实现，要么删除。
- **敏感信息：** 禁止在代码中硬编码密钥、密码等敏感信息，使用环境变量或配置文件。
- **输入验证：** 所有用户输入必须验证，使用 `validator` crate。
- **错误处理：** 禁止使用 `unwrap()` 或 `expect()` 处理可能失败的操作（测试代码除外）。
- **代码操作：** 生成或修改代码时避免逻辑混乱及幽灵代码。
- **错误处理：** 所有可能失败的操作都必须有对应的错误处理代码。 
- 项目使用 edition = "2024" 版本，确保代码符合最新的Rust语言规范，且代码优雅
- rust 版本0.xx.yy 每次修改代码更新yy一次
