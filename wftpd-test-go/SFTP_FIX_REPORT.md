# WFTPD-Test-Go SFTP 连接问题修复报告

## 问题描述
wftpd-test-go 程序连接 SFTP 服务器时存在问题，而 WinSCP 连接正常。

## 发现的问题

### 1. 编译错误（已修复）
**位置**: main.go 第 412-475 行 `testDirectoryOperations` 函数
**问题**: 定义了变量 `fc`，但后续代码使用了未定义的变量 `c`
**修复**: 将所有 `c.` 替换为 `fc.`

### 2. SFTP 客户端关闭超时（已修复）
**位置**: sftp.go 第 70-86 行 `Close()` 方法
**问题**: 
- Go sftp 库在调用 `Client.Close()` 时会等待所有未完成的请求完成
- SFTP 服务器可能没有正确处理某些清理请求
- 导致客户端无限等待，测试程序卡住约 90 秒

**修复**: 
- 在 `Close()` 方法中添加 5 秒超时机制
- 使用 goroutine 和 channel 实现异步关闭
- 如果超时则强制继续，避免阻塞后续测试

```go
// 使用 goroutine 和 channel 来实现超时关闭
done := make(chan error, 1)
go func() {
    done <- c.Client.Close()
}()

select {
case err := <-done:
    // 正常关闭
case <-time.After(5 * time.Second):
    logger.Printf("  [DEBUG] SFTP 客户端关闭超时，强制关闭\n")
}
```

### 3. SFTP 连接在操作中断开（待优化）
**现象**: 
- 部分测试出现 "connection lost" 错误
- 特别是文件上传/下载操作

**可能原因**:
- SFTP 服务器实现与 Go sftp 库的期望行为不完全兼容
- 数据包格式或协议版本协商问题
- 并发请求处理问题
