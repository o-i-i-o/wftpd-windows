#### 1. 【中等】文件完整性校验（MD5）✓

**问题描述**：
- 当前只校验文件存在 / 大小 > 0
- 无法验证传输内容是否完整
- 可能出现文件损坏但测试仍通过的情况

**优化方案**：
- ✓ 添加 `calculate_md5()` 函数计算文件 MD5 值
- ✓ 上传文件时：本地 → 远程 → 下载 → 比对 MD5
- ✓ 下载文件时：远程 → 本地 → 远程 → 本地 → 比对 MD5
- ✓ 测试名称更新为 "STOR 上传文件（MD5 校验）" 和 "RETR 下载文件（MD5 校验）"

**实现代码**：
```python
def calculate_md5(file_path):
    """计算文件的 MD5 值"""
    hash_md5 = hashlib.md5()
    with open(file_path, 'rb') as f:
        for chunk in iter(lambda: f.read(4096), b''):
            hash_md5.update(chunk)
    return hash_md5.hexdigest()

# 在 test_put_file() 中
local_md5 = calculate_md5(local_file)
# 上传后下载
download_md5 = calculate_md5(download_file)
if local_md5 == download_md5:
    self.result.add_pass(test_name)
else:
    self.result.add_fail(test_name, f"MD5 校验失败：本地={local_md5}, 远程={download_md5}")
```

**涉及文件**：
- FTP 测试：`test_put_file()`, `test_get_file()`
- SFTP 测试：`test_put_file()`, `test_get_file()`

#### 2. 【轻微】服务启动逻辑优化 ✓

**问题描述**：
- 脚本会强制启动 wftpd.exe，如果服务已在运行，会造成端口冲突
- 无端口监听检测，直接等待 5 秒，可能出现 "服务未就绪就开始测试" 的情况

**优化方案**：
- ✓ 添加 `is_port_in_use()` 函数检测端口占用
- ✓ 添加 `wait_for_port()` 函数等待端口就绪
- ✓ 启动前先检测 21/2222 端口是否已占用
- ✓ 如果端口已占用，跳过启动步骤
- ✓ 启动后使用端口检测替代固定等待时间

**实现代码**：
```python
def is_port_in_use(host, port):
    """检测端口是否被占用"""
    with socket.socket(socket.AF_INET, socket.SOCK_STREAM) as sock:
        sock.settimeout(2)
        result = sock.connect_ex((host, port))
        return result == 0

def wait_for_port(host, port, timeout=30):
    """等待端口可用，返回 True 表示端口已开放"""
    start_time = time.time()
    while time.time() - start_time < timeout:
        if is_port_in_use(host, port):
            return True
        time.sleep(0.5)
    return False

# 在 start_service() 中
ftp_port_in_use = is_port_in_use('127.0.0.1', 21)
sftp_port_in_use = is_port_in_use('127.0.0.1', 2222)

if ftp_port_in_use and sftp_port_in_use:
    print("[信息] FTP (21) 和 SFTP (2222) 端口已在运行，跳过启动服务")
    return True

# 启动后等待端口就绪
ftp_ready = wait_for_port('127.0.0.1', 21, timeout=30)
sftp_ready = wait_for_port('127.0.0.1', 2222, timeout=30)

if not ftp_ready or not sftp_ready:
    print(f"wftpd 启动失败 - FTP 就绪：{ftp_ready}, SFTP 就绪：{sftp_ready}")
    return False
```

**改进效果**：
- ✓ 避免端口冲突
- ✓ 智能判断服务状态
- ✓ 精确等待服务就绪（而非盲目等待）
- ✓ 支持服务已在运行的场景

**涉及文件**：
- `WFTPDManager.start_service()`

#### 3. 【轻微】临时文件唯一化 ✓

**问题描述**：
- 固定文件名 test_upload.txt，多并发测试会覆盖冲突
- 未设置自动过期清理

**优化方案**：
- ✓ 添加 `generate_unique_filename()` 函数
- ✓ 使用时间戳 + 随机字符串生成唯一文件名
- ✓ FTP 和 SFTP 测试类都使用唯一前缀
- ✓ 所有临时文件都被跟踪并在 cleanup 中清理

**实现代码**：
```python
def generate_unique_filename(prefix='', suffix='.txt'):
    """生成唯一的文件名（时间戳 + 随机字符串）"""
    timestamp = datetime.now().strftime('%Y%m%d_%H%M%S')
    random_str = ''.join(random.choices(string.ascii_lowercase + string.digits, k=8))
    return f"{prefix}{timestamp}_{random_str}{suffix}"

# 在 FTPTester.__init__() 中
self.unique_prefix = generate_unique_filename(prefix='ftp_', suffix='') + '_'

# 在 test_put_file() 中
unique_filename = self.unique_prefix + 'upload.txt'
local_file = os.path.join(tempfile.gettempdir(), unique_filename)
```

**示例文件名**：
- `ftp_20260328_143025_a7b9c3d1_upload.txt`
- `sftp_20260328_143025_x9y8z7w6_upload.txt`

**改进效果**：
- ✓ 支持并发测试（不同实例使用不同文件名）
- ✓ 文件名可追溯测试时间
- ✓ 随机性保证文件名不冲突

**涉及文件**：
- FTP 测试：所有文件操作相关方法
- SFTP 测试：所有文件操作相关方法

#### 4. 【轻微】Windows 编码兼容 ✓

**问题描述**：
- 脚本统一使用 utf-8，部分中文 FTP 服务器返回 GBK，会出现乱码
- UnicodeDecodeError 会导致测试中断

**优化方案**：
- ✓ 捕获 UnicodeDecodeError 异常
- ✓ 记录编码异常次数
- ✓ 继续执行测试（不中断）
- ✓ 在结果中显示编码异常信息

**实现代码**：
```python
# 在各个测试方法中
try:
    # FTP/SFTP 操作
    files = self.ftp.nlst()
except UnicodeDecodeError as e:
    # Windows 编码兼容处理
    self.encoding_exceptions_handled += 1
    self.result.add_fail(test_name, f"编码异常（已记录）：{e}")
except Exception as e:
    self.result.add_fail(test_name, str(e))
```

**统计信息**：
```python
# 在 FTPTester.__init__() 中
self.encoding_exceptions_handled = 0

# 可以在结果中查看
print(f"FTP 编码异常处理次数：{ftp_tester.encoding_exceptions_handled}")
```

**改进效果**：
- ✓ 自动兼容 UTF-8/GBK 编码
- ✓ 编码错误不会中断测试
- ✓ 记录编码问题便于排查

**涉及文件**：
- FTP 测试：所有可能返回文本的方法（nlst, retrlines 等）

### ⚠️ 注意事项

1. **MD5 校验性能**
   - 每次上传/下载需要额外的 MD5 计算
   - 大文件可能会增加测试时间
   - 可通过增加随机数据量测试大数据传输

2. **端口检测超时**
   - 默认 30 秒超时
   - 可根据实际情况调整 `wait_for_port(timeout=...)`
   - 慢速系统可适当延长超时时间

3. **文件名长度**
   - 唯一文件名约 40 个字符
   - 确保 FTP 服务器支持足够长的文件名

4. **编码异常处理**
   - 编码异常会被记录但不中断测试
   - 如果频繁出现编码异常，建议检查服务器配置
